# Driving Companion App

## 概要

運転中に、結月ゆかりや紲星あかりなどの音声が助手席から話しかけてくれるような体験を実現する個人向けアプリ。

CarPlay接続中に一定間隔で音声を再生し、長距離運転時の眠気対策やドライブの楽しさ向上を目的とする。

---

## 実現したい体験

- SpotifyやApple Musicで音楽を流しながら運転
- 30分ごとにキャラクターが話しかける
- ナビ音声のようにBGMを少し下げて再生
- 再生後は自動的に元の音量へ戻る
- ランダムなセリフで飽きない
- CarPlay接続中のみ動作

例:

> 「そろそろ休憩しませんか？」
>
> 「次のSAまであと少しですよ」
>
> 「眠そうですね、大丈夫ですか？」

## 技術検討の経緯

### ショートカットアプリ

検討した内容:

- CarPlay接続をトリガー
- 30分ごとに音声再生

問題点:

- CarPlay接続時は1回しか実行されない
- 長時間のWaitが不安定
- バックグラウンドで停止する可能性が高い

結論:

- 実用性は低い

---

### Webアプリ（PWA）

検討した内容:

- TypeScript/Vueで実装
- ローカルMP3再生
- 30分タイマー

メリット:

- 開発しやすい
- 既存スキルを活用できる
- GitHub Pagesなどで公開可能

問題点:

- iOSのバックグラウンド制限
- Safariのタイマー停止
- Audio APIの制約
- Spotify等との共存が不安定

結論:

- PoCには向いている
- 本命にはなりにくい

---

### ネイティブアプリ（Swift）

検討した内容:

- SwiftUI
- AVFoundation
- AVAudioSession

メリット:

- バックグラウンド再生対応
- Audio Ducking対応
- CarPlay接続状態取得
- 安定動作

結論:

- 最終的な本命

## 開発環境の課題

現在:

- Macなし
- iPadあり
- TypeScript / Vue / FastAPI経験あり
- Swift経験なし

課題:

- XcodeがMac必須
- iOS実機ビルドにMacが必要

検討案:

1. Vue/PWAでPoC作成
2. 動作検証
3. Mac入手後Swift化

---

## 開発方針

優先順位:

1. バックグラウンド動作
2. Audio Ducking
3. CarPlay連携
4. 音声再生機能
5. UI

UIよりも音声周りの安定性を重視する。
